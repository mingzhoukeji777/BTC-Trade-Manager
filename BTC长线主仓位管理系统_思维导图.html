BTC LONG-CYCLE POSITION MANAGEMENT SYSTEM

BTC 长线主仓位管理系统思维导图

正确主线：Binance U本位永续 + OKX 币本位永续都属于长线牛熊周期账户；系统不是短线/波段/长线账户分组，而是围绕“主仓位”管理加仓、对冲、减仓、移动止损、移动止盈、分批止盈与风险联动。

橙色 = 核心主线绿色 = 规则/收益保护红色 = 风险监控灰色 = 数据/记录

我推荐的软件主界面结构

  • 左侧树 牛熊周期 → 主仓位 → 加仓节点 / 对冲节点 / 减仓节点 / 止盈节点。树上每个节点都显示状态颜色。
  • 中间总控 主仓成本、实时价格、强平价、强平距离、账户权益、保证金占用、总浮盈亏。
  • 右侧明细 选中某个加仓/对冲节点后，显示成本、手续费、盈亏平衡、成本止损、移动止盈、推荐动作。
  • 底部风险条 红黄绿风险等级、距离触发价、距离强平价、是否需要减仓/移动止盈。

关键计算对象

对象必须计算
主仓位主仓均价、数量、保证金、强平、浮盈亏、周期方向
加仓部分加仓成本、双向手续费、盈亏平衡价、成本止损、移动止盈
对冲部分对冲成本、手续费、盈亏平衡、解锁/平仓条件、对主仓保护效果
减仓/止盈已实现盈亏、剩余主仓均价变化、风险释放程度
风险系统强平距离、可承受回撤、保证金压力、节点触发距离

需要改掉之前 v0.1 框架的地方

  • 账户模型：不再是“短线/波段/长线账户”，而是 Binance U本位长线账户OKX 币本位长线账户。
  • 核心对象：从“交易计划列表”改成“周期主仓位 + 操作节点树”。
  • 计划逻辑：每次加仓/对冲/减仓不是孤立记录，必须挂在某个主仓或父节点下面。
  • 风险逻辑：所有操作都要实时反算对主仓的影响，而不是只展示当前交易所持仓。

不足 / 待确认 / 我建议补充

  • 必须确认 Binance U本位和 OKX 币本位是否分别独立管理主仓，还是汇总成一个“全局BTC主仓风险”。我建议两层都做：单账户 + 汇总。
  • 必须确认 加仓和对冲的手续费率、滑点、资金费率是否按交易所实时值估算，还是手动配置固定值。
  • 建议 每个操作节点必须有唯一编号：主仓 M1，加仓 A1/A2，对冲 H1/H2，止盈 TP1/TP2，避免后续混乱。
  • 建议 增加“情景模拟”：如果 BTC 跌/涨到某价格，主仓、加仓、对冲、强平距离分别变成多少。
  • 建议 增加“动作建议不是自动交易”：系统只提示移动止盈/止损/减仓建议，不自动下单，避免误操作。
  • 建议 Excel 同步只做报表和备份，核心计算在 C#，避免 Excel 公式复杂失控。
