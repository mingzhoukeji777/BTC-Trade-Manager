# BTC交易管理系统

当前版本：`v0.4.0 主仓树整页修正版`

## 核心定位

这是围绕完整牛熊周期设计的 BTC 长线主仓位管理系统。

分别管理：

```text
Binance U本位永续独立主仓
OKX 币本位永续独立主仓
```

两个账户独立管理，通过顶部下拉框切换。

## v0.4 修复内容

### 1. 去掉标题栏多余文字

窗口标题不再显示多余的“主仓树”。

### 2. 修复表格选中行自动跳回第一行

之前定时刷新会重载所有表格，导致你选中第二行后几秒又跳回第一行。

现在定时刷新只更新：

```text
总控
风险总控
采集器实时数据
```

不会重载主仓表、节点表、订单表，所以不会打断你的选择。

手动点“刷新”才会完整刷新所有页面。

### 3. 周期改为下拉菜单

主仓添加/修改窗口中，周期为下拉：

```text
牛市周期
熊市周期
```

### 4. 去掉添加窗口里的无用 0

主仓、节点、订单添加窗口中，数量、价格、手续费、资金费等字段默认留空，不再填一堆 0。

### 5. 独立添加模块

顶部按钮已拆分为：

```text
加仓
减仓
对冲
移动止盈
成本止损
```

不再只用一个“新增节点”混合入口。

节点明细页面右键也有：

```text
新增加仓
新增减仓
新增对冲
新增移动止盈
新增成本止损
修改节点
删除节点
```

### 6. 下拉菜单增强

以下字段已使用下拉菜单：

```text
主仓编号
节点编号
方向
状态
节点类型
订单方向
订单动作
订单匹配状态
```

## 建议测试流程

### 1. 测试主仓管理

1. 打开“主仓管理”页。
2. 右键新增主仓。
3. 周期选择“牛市周期”或“熊市周期”。
4. 方向选择“多”或“空”。
5. 保存后检查是否出现在表格里。
6. 选中该行，右键修改。
7. 选中该行，右键删除。

### 2. 测试独立节点入口

分别点击顶部：

```text
加仓
减仓
对冲
移动止盈
成本止损
```

检查：

```text
标题是否对应
节点编号是否自动给对应编号段
类型是否自动对应
数量/价格/手续费是否为空白
保存后是否进入“节点明细”和“主仓树”
```

### 3. 测试选中行不跳动

1. 进入“主仓管理”。
2. 选中第二行。
3. 等 10 秒。
4. 如果仍停在第二行，说明修复成功。

### 4. 测试订单映射

1. 进入“订单映射/待归类”。
2. 右键新增订单。
3. 匹配状态选“待归类”。
4. 保存后检查表格。
5. 右键修改为“已匹配”。
6. 再测试删除。

### 5. 测试窗口保存

1. 调整窗口大小和位置。
2. 关闭软件。
3. 重新打开。
4. 检查是否恢复上次大小和位置。
