# BTC交易管理系统

当前版本：`v0.7.6 订单归类增强版`

## v0.7.6 新增/增强

### 1. 归类时支持自动新建 A/H/R 目标

在 `订单映射/待归类` 页面右键订单：

```text
归类到加仓A
归类到对冲H
归类到减仓R
```

现在目标下拉里会出现：

```text
新建 M1-A3
新建 M1-H2
新建 M1-R1
```

选择后会自动创建对应节点，并回填订单数据：

```text
价格
数量
手续费
开仓时间
开仓订单号
备注
状态 = 持有中
```

### 2. 归类仍支持已有目标

如果已有：

```text
M1-A1
M1-H1
M1-R1
```

也可以直接选择已有目标，系统会更新对应记录。

### 3. 资金费自动汇总

归类为资金费后，现在会：

```text
写入 funding_records
同步更新主仓 cumulative_funding
或同步更新节点 cumulative_funding
```

### 4. 备注自动保留原始订单信息

归类后备注会追加：

```text
原始订单:
订单号
成交号
动作
方向
价格
数量
手续费
资金费
时间
```

方便后续复盘追溯。

### 5. 新增撤销归类

订单右键新增：

```text
撤销归类
```

当前版本撤销逻辑：

```text
订单状态改回 待归类
清理订单备注中的“已归类到...”标记
暂不反向删除/修改目标记录，避免误删真实持仓数据
```

## 当前限制

```text
1. 自动新建支持 A/H/R，主仓仍需先建立。
2. 撤销归类目前只撤销订单状态，不反向删除目标记录。
3. 资金费已汇总到 cumulative_funding，但还没有做资金费展示专页。
4. 减仓影响主仓剩余数量/成本的精确逻辑还没做。
```
