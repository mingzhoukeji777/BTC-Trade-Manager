# BTC交易管理系统

当前版本：`v0.7.2 表格列宽锁定/表头居中/主控放大版`

## v0.7.2 修复/新增

### 1. 修改表格内容后不再自动改变列宽

之前没有保存过列宽的表格，在刷新数据后会按内容重新自动适应，导致修改内容后列宽变化。

现在改为：

```text
有保存列宽：严格使用保存列宽
没有保存列宽：只按标题栏文字生成默认宽度
不再按内容自动撑宽
```

### 2. 表头真正居中

修复 WinForms 默认表头视觉样式覆盖居中的问题。

现在所有表格统一：

```text
EnableHeadersVisualStyles = false
表头居中
表格内容居中
格式化盈亏颜色后仍保持居中
```

### 3. 所有表格内容字体加大

普通表格内容字体从 9 号加大到 11 号，行高加高。

主控页内容字体继续加大：

```text
主控内容：20号加粗
主控表头：15号加粗
主控行高：66
```

### 4. 主控手动列宽不再被切标签覆盖

移除了切换标签时强制按比例重置主控列宽的逻辑。

现在主控页：

```text
手动调整列宽后立即保存
切换标签回来使用保存列宽
不再强制恢复比例宽度
```

### 5. 双击列分隔符仍按标题栏文字收紧

双击列分隔符后：

```text
按标题栏文字宽度收紧
立即保存列宽
重新应用表头和内容居中
```

### 6. 本地 EXE 说明

由于你当前打开着旧版程序，Windows 锁住了：

```text

```

所以本次没有强行关闭旧程序，也没有覆盖正在运行的 EXE。

新版已另存为：

## 订单映射/待归类功能说明

当前版本的“订单映射/待归类”是手动归类台账，用来记录交易所成交单和管理系统里主仓/加仓/对冲/减仓动作之间的关系。

主要字段：

```text
订单号：交易所订单 ID
成交号：交易所成交/Trade ID
方向：多 / 空 / 买入 / 卖出
动作：开仓 / 平仓 / 加仓 / 对冲 / 止盈 / 止损 / 减仓 / 资金费
价格：成交价格
数量：成交数量
手续费：该笔成交手续费
资金费：资金费记录时填写
时间：成交时间
匹配状态：已匹配 / 待归类 / 忽略
备注：手工说明
```

当前用法：

1. 点击顶部 `归类订单` 或进入 `订单映射/待归类` 页面右键新增订单。
2. 从交易所成交记录复制订单号、成交号、价格、数量、手续费等。
3. 选择动作，例如：加仓、对冲、平仓、减仓、资金费。
4. 如果还没确认归属，匹配状态选 `待归类`。
5. 确认这笔成交已经对应到某个主仓/加仓/对冲/减仓后，改为 `已匹配`。
6. 明显不需要处理的记录改成 `忽略`。

注意：当前它还没有自动把订单映射到具体 A/H/R 编号，只是先做“待归类台账”。后续可以继续升级为：选择订单后右键归类到 M/A/H/R，并自动回填手续费、平仓价格和资金费。
