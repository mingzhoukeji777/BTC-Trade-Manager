# BTC交易管理系统

当前版本：`v0.6.2 交互修复版`

## v0.6.2 修复/新增

### 1. 保存 / 取消按钮位置对换

所有通用弹窗底部按钮已对换位置。

### 2. 新增仓位默认状态改为“已开仓”

主仓、加仓、对冲、减仓新增时，状态默认优先为：

```text
已开仓
```

### 3. OKX / Binance 默认单位

新增/修改窗口按当前账户显示单位：

```text
OKX 币本位主仓：BTC
Binance U本位主仓：USDT
```

例如：

```text
OKX：主仓数量(BTC)、保证金(BTC)、数量(BTC)
Binance：主仓数量(USDT)、保证金(USDT)、数量(USDT)
```

### 4. 新增主仓不自动计算手续费和盈亏平衡

主仓手续费 / 盈亏平衡价不再自动计算，后续以真实订单/API回填为准。

### 5. 加仓 / 对冲 / 减仓表格去掉周期

三个分支模块不再显示周期列。

周期只保留在主仓层级。

### 6. 增加“已平仓”模块

新增独立页面：

```text
已平仓
```

状态为：

```text
已平仓
```

的加仓 / 对冲 / 减仓会集中显示到这里。

未平仓页面会过滤掉已平仓记录。

### 7. 主仓周期简化

主仓周期下拉从：

```text
牛市周期 / 熊市周期
```

简化为：

```text
牛 / 熊
```

旧数据仍兼容显示。

### 8. 加仓 / 对冲 / 减仓编号扩展到 1000

编号范围：

```text
加仓：A1 - A1000
对冲：H1 - H1000
减仓：R1 - R1000
```

新增时继续自动推荐下一个未使用编号。

### 9. 修复“修改对冲状态”唯一约束错误

修复原因：编辑 H 节点时，编号下拉曾错误使用 A 编号列表，导致保存时可能把 H1 改成 A1，从而触发：

```text
UNIQUE constraint failed: position_nodes.account_key, position_nodes.main_code, position_nodes.code
```

现在编辑已有节点会按原节点类型加载对应编号列表，并默认选中原编号。

## 当前测试重点

1. 右键修改 H1 对冲，把状态改成已开仓/持有中/已平仓，保存不应报唯一约束错误。
2. 平仓后的节点应出现在“已平仓”页，并从加仓/对冲/减仓当前页过滤掉。
3. 新增加仓/对冲/减仓时，默认状态应是“已开仓”。
4. OKX 下新增时单位应显示 BTC；Binance 下新增时单位应显示 USDT。
5. 加仓/对冲/减仓表格不应再有周期列。
