# BTC交易管理系统

当前版本：`v0.2.0 主仓树框架版`

## 核心定位

这是围绕完整牛熊周期设计的 BTC 长线主仓位管理系统。

它不是短线/波段/长线账户分类工具，而是：

```text
Binance U本位永续独立主仓
OKX 币本位永续独立主仓
```

两个账户独立管理，通过顶部下拉框切换。

## v0.2 已实现

### 1. Binance / OKX 独立主仓

默认初始化：

```text
BINANCE_UM / M1 / 多 / 规划中
OKX_COIN  / M1 / 多 / 规划中
```

### 2. 主仓树结构

左侧树状结构：

```text
M 主仓
├ A 加仓
├ H 对冲
├ TP 止盈
├ SL 止损
└ R 减仓
```

每个加仓/对冲都按独立小主仓位分支处理。

### 3. 节点字段

节点支持：

```text
编号
类型
方向
状态
数量
开仓价
手续费
资金费
盈亏平衡价
成本止损
移动止盈
交易所开仓订单号
交易所平仓订单号
备注
```

### 4. 订单映射 / 待归类订单

支持手动新增订单映射：

```text
订单号
成交号
方向
动作
价格
数量
手续费
资金费
时间
匹配状态：已匹配 / 待归类
备注
```

后续 API 自动拉订单后，如果无法识别是加仓/对冲/止盈/止损，会进入这个模块手动归类。

### 5. 手续费规则

默认：

```text
0.05% 单向
```

新增节点时，如果不填手续费，会按：

```text
数量 * 开仓价 * 0.05%
```

自动估算。

后续通过 API 获取真实手续费后，真实手续费会覆盖默认估算。

### 6. 资金费率

已建立 `funding_records` 表，用于后续接入 API 每次结算资金费。

资金费将归入：

```text
对应交易所主仓
对应加仓/对冲分支
```

### 7. 情景模拟

当前支持输入目标 BTC 价格，生成一条情景模拟记录。

后续会扩展为：

```text
目标价下主仓盈亏
加仓分支盈亏
对冲分支盈亏
强平距离变化
是否建议移动止盈/减仓/解除对冲
```

### 8. 采集器数据读取

继续读取：

```text
C:\Users\xiaozhu\Desktop\BTC实时通信系统\data\latest_snapshot.json
```

用于显示当前价格、权益、持仓、浮盈亏、强平价。

### 9. UI 布局自动保存

手动调整后会自动保存：

```text
窗口大小
窗口位置
左右分栏宽度
```

下次打开自动恢复。

配置文件：

```text
ui_settings.json
```
