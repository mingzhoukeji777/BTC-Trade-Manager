# BTC交易管理系统

当前版本：`v0.5.0 主仓树模块拆分版`

## v0.5 修复/新增

### 1. 编号扩展 + 自动下一个编号

编号数量已扩展，覆盖完整牛熊周期：

```text
主仓：M1 - M50
加仓：A1 - A100
对冲：H1 - H100
减仓：R1 - R100
移动止盈：TP1 - TP100
成本止损：SL1 - SL100
```

新增时会自动优先选择下一个未使用编号。

例如已有：

```text
A1 A2 A3 A4 A5
```

下次新增加仓时默认显示：

```text
A6
```

### 2. 节点明细拆成独立模块

已拆分为独立页面：

```text
加仓
对冲
减仓
```

主仓树只显示：

```text
加仓
对冲
```

因为加仓和对冲是挂在主仓下面的核心分支；减仓直接影响主仓数量和风险，但不作为树上长期持有分支展示。

### 3. 成本止损 / 移动止盈改为右键保护动作

顶部按钮已去掉：

```text
移动止盈
成本止损
```

现在只对持有中的加仓/对冲节点生效。

使用方式：

```text
选中持有中的加仓或对冲节点
右键
添加成本止损 / 添加移动止盈
填写价位
保存后节点状态自动变成 成本止损 / 移动止盈
```

### 4. 加仓/对冲窗口优化

新增加仓/对冲窗口中：

```text
平仓价 -> 止损价
去掉资金费
去掉成本止损
去掉移动止盈
```

保留：

```text
数量
开仓价
止损价
手续费
盈亏平衡
开仓订单号
平仓订单号
备注
```

手续费和盈亏平衡可以留空，保存时会自动计算：

```text
手续费 = 数量 * 开仓价 * 0.05%
多单盈亏平衡 = 开仓价 * (1 + 双向手续费率)
空单盈亏平衡 = 开仓价 * (1 - 双向手续费率)
```

### 5. 主仓窗口补单位和详细信息

主仓添加/修改窗口字段：

```text
主仓数量(BTC/张)
主仓均价(USDT/USD)
硬止损价(USDT/USD)
保证金(USDT/BTC)
手续费(USDT/BTC)
盈亏平衡价
开仓订单号
备注
```

### 6. 顶部按钮顺序调整

现在顺序：

```text
新增主仓
加仓
对冲
减仓
归类订单
情景模拟
设置
```

## 测试建议

### 编号测试

1. 连续新增加仓 A1-A5。
2. 再点“加仓”。
3. 应默认显示 A6。

### 持有保护测试

1. 新增加仓，状态选“已开仓”或“持有中”。
2. 保存后进入“加仓”页。
3. 选中该行右键。
4. 添加成本止损。
5. 检查状态是否变成“成本止损”。
6. 再添加移动止盈，检查状态是否变成“移动止盈”。

### 主仓树测试

1. 新增加仓。
2. 新增对冲。
3. 新增减仓。
4. 打开“主仓树”。
5. 应只看到加仓和对冲，不显示减仓。
