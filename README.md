# BTC交易管理系统

当前版本：`v0.7.5 订单映射归类功能版`

## v0.7.5 新增

### 1. 订单映射右键归类菜单

在 `订单映射/待归类` 页面右键订单，新增：

```text
归类到主仓M
归类到加仓A
归类到对冲H
归类到减仓R
归类为平仓
归类为资金费
标记已匹配
标记忽略
标记待归类
```

### 2. 归类到主仓M

选择目标主仓后，会自动回填：

```text
主仓均价
主仓数量
手续费
开仓时间
开仓订单号
状态 = 持有中
备注
```

订单状态自动改为：

```text
已匹配
```

### 3. 归类到加仓A / 对冲H / 减仓R

选择目标编号，例如：

```text
M1-A3
M1-H2
M1-R1
```

会自动回填：

```text
开仓价
数量
手续费
开仓时间
开仓订单号
状态 = 持有中
备注
```

订单状态自动改为：

```text
已匹配
```

### 4. 归类为平仓

可选择目标：

```text
主仓:M1
加仓:M1-A3
对冲:M1-H2
```

会自动回填：

```text
平仓价格
平仓类型
平仓时间
平仓订单号
手续费
状态 = 已平仓
浮动/已实现盈亏
备注
```

订单状态自动改为：

```text
已匹配
```

### 5. 归类为资金费

可选择目标主仓或节点。

会写入：

```text
funding_records
```

用于后续累计资金费统计。

### 6. 快速标记状态

右键订单可以直接标记：

```text
已匹配
忽略
待归类
```

## 当前限制

本版是第一版“归类功能”，仍有几个限制：

```text
1. 目标记录需要先存在，暂不支持归类时自动新建 A/H/R。
2. 归类会写回目标记录，但减仓对主仓剩余成本/数量的精确影响还没做。
3. 平仓盈亏是基础公式，后续还需要专门审计 OKX 币本位 / Binance U本位 的单位和手续费逻辑。
4. 资金费已写入 funding_records，但还未自动汇总到主仓/节点累计资金费字段。
```
