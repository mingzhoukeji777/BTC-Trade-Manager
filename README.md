## v0.7.4 修复/新增

### 1. 风险总控布局优化

根据截图，原风险总控页面表格只挤在左上角，右侧大面积空白，结果/说明列显示不全。

本版改为：

```text
风险总控横向铺满
新增 分类 / 项目 / 结果 / 说明 四列
说明列自动占用剩余宽度
字体 13 号
行高 44
内容自动换行
表头和内容居中
```

### 2. 风险总控新增风险结论

新增第一行：

```text
风险结论
```

会根据数据给出：

```text
正常
待处理
警戒
危险
无数据
```

并用颜色区分：

```text
正常：绿色
待处理/警戒：橙色
危险：红色
```

### 3. 风险总控新增结构/保护统计

新增展示：

```text
活跃主仓
活跃节点
总节点数量
成本止损节点
移动止盈节点
待归类订单
```

比原来只显示少量数据更适合做风控检查。

## 当前功能后续优化建议

### 优先级 A：订单映射升级为真正归类功能

当前“订单映射/待归类”还是手动台账。建议下一步做：

```text
右键订单 -> 归类到 主仓M / 加仓A / 对冲H / 减仓R
选择目标编号，例如 M1-A3
自动把订单手续费/价格/数量/时间写回目标记录
自动把匹配状态改为 已匹配
```

这是现在最值得做的功能，因为它能让成交记录、手续费、平仓价格、盈亏统计真正串起来。

### 优先级 B：风险总控继续做成分区看板

下一步可以把风险总控从表格升级为：

```text
顶部：风险状态卡片
中部：账户/价格/强平/持仓
底部：待归类订单/保护状态/异常提醒
```

会比表格更直观。

### 优先级 C：盈亏和手续费计算统一审计

目前主仓、加仓、对冲、减仓、已平仓都已有字段，但建议后面做一次底层公式审计：

```text
OKX 币本位：手续费单位 BTC
Binance U本位：手续费单位 USDT
多/空盈亏方向
减仓对主仓剩余成本的影响
平仓后已实现盈亏
累计手续费/资金费
```

### 优先级 D：已平仓复盘模块

已平仓现在能修改字段，下一步建议加：

```text
平仓原因
是否按计划执行
错误类型
复盘结论
下次改进
```

后面可以生成交易复盘报告。

### 优先级 E：自动备份数据库

建议每次启动或每天自动备份：

```text
data/btc_trade_manager.sqlite3
```

避免手工测试时误删数据。
