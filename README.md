# BTC交易管理系统

当前版本：`v0.6.1 DataGridView显示错误修复版`

## v0.6.1 修复

修复 v0.6 表格显示时可能弹出的 DataGridView 默认错误：

```text
Invalid cast from System.String to System.Drawing.Image
```

处理方式：

```text
所有表格统一强制为文本列
增加 DataGridView.DataError 保护
浮动收益列单独做颜色格式化，不再触发表格类型转换错误
```

## v0.6 修复/新增

### 1. “主仓位”统一更名为“主仓”

界面用语统一为：

```text
主仓
加仓
对冲
减仓
```

### 2. 主仓 / 加仓 / 对冲统一表格格式

主仓、加仓、对冲现在统一使用接近一致的字段：

```text
编号
方向
周期
状态
持仓量
开仓均价
盈亏平衡价
强平价格
浮动收益
维持保证金率
保证金
止损价格
成本止损
移动止盈
平仓价格
开仓订单号
平仓订单号
备注
```

其中浮动收益列支持颜色：

```text
浮盈：绿色
浮亏：红色
```

### 3. 主仓右键菜单简化

主仓右键只保留：

```text
修改
删除
移动止盈
成本止损
平仓
```

### 4. 加仓 / 对冲右键菜单简化

加仓、对冲、减仓页面右键也只保留：

```text
修改
删除
移动止盈
成本止损
平仓
```

其中移动止盈、成本止损主要用于持有中的加仓/对冲节点。

### 5. 平仓功能

主仓、加仓、对冲都支持右键：

```text
平仓
```

填写：

```text
平仓价格
平仓订单号
```

保存后：

```text
平仓价格写入表格
订单状态自动切换为 已平仓
已平仓条目从主仓树中移除
```

### 6. 主仓树行为修复

主仓树现在：

```text
鼠标双击不会再弹出修改界面
只有右键菜单才能修改/删除/保护/平仓
已平仓的加仓/对冲不会显示在主仓树中
```

主仓树只显示：

```text
加仓
对冲
```

### 7. 手续费 / 盈亏平衡实时自动计算

在加仓/对冲/减仓添加窗口中，填写：

```text
数量
开仓价
```

后，系统会实时自动填入：

```text
手续费(自动)
盈亏平衡(自动)
```

保存时也会再次计算，避免漏填。

默认手续费：

```text
0.05% 单向
```

### 8. 总控界面优化

总控页从纯文本改成表格排版，字段包括：

```text
当前主仓
更新时间
状态
实时价格
标记价
强平价
持仓
浮动收益
强平距离
```

视觉上比原来更清楚。

### 9. 顶部页面分隔增强

Tab 页签改成带边框绘制，分隔比之前更清楚。

## 当前测试重点

### 测试 1：手续费/盈亏平衡自动计算

1. 点击“对冲”或“加仓”。
2. 数量填 `1`。
3. 开仓价填 `60000`。
4. 检查手续费和盈亏平衡是否自动出现。

### 测试 2：平仓

1. 在加仓或对冲页面选中持有中订单。
2. 右键选择“平仓”。
3. 填写平仓价格。
4. 保存。
5. 检查状态是否变成“已平仓”。
6. 打开主仓树，确认该条目不再显示。

### 测试 3：主仓保护动作

1. 主仓管理页选中主仓。
2. 右键选择“成本止损”或“移动止盈”。
3. 填写价位。
4. 检查表格中成本止损/移动止盈列是否写入。

### 测试 4：树双击

1. 打开主仓树。
2. 双击任意节点。
3. 不应弹出修改窗口。
4. 右键才应出现修改/删除/平仓等菜单。
